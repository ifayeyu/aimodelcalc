How AI pricing works

AI APIs charge per token — roughly ¾ of a word. Most models charge separately for input (your prompt) and output (the response). Output tokens are typically 3–5× more expensive than input tokens.

How AI API Pricing Works: A Plain-English Guide

AI APIs like GPT-4o, Claude, and Gemini don't charge a flat monthly fee — they charge per token, where a token is roughly 0.75 words (or about 4 characters) of text. This means costs scale directly with how much text you send and receive. Understanding this model is essential for anyone building on top of AI APIs, because costs can range from fractions of a cent for a short query to hundreds of dollars per day for a high-volume production application.

Every AI API request has two components: the prompt (input tokens) — everything you send to the model, including the system prompt, conversation history, and user message — and the completion (output tokens) — the text the model generates in response. Output tokens are almost always more expensive than input tokens, because generating text is computationally more intensive than reading it.

Input vs. Output Tokens: Why the Ratio Matters

The split between input and output tokens in your application dramatically affects your cost per request. Consider two common use cases:

  • Document summarization — You send a large document (many input tokens) and ask for a short summary (few output tokens). Input-heavy. At GPT-5.4's pricing of $2.50/M input vs $15/M output, the low output ratio keeps costs manageable even for long documents.
  • Content generation — You send a short prompt but receive a long, detailed response. Output-heavy. The same GPT-5.4 can become significantly more expensive because output is 6× the cost of input.
  • Chat applications with long history — Every turn in a multi-turn conversation re-sends the entire conversation history as context. As conversations grow, input token costs compound quickly — a 20-turn conversation can cost 10× more per turn than a fresh one-shot query.

Choosing the Right Model for Your Use Case

The most expensive model is rarely the right choice for every task. AI API pricing spans several orders of magnitude — from sub-$0.10/M token models like Gemini Flash-Lite to $25–30/M token models like GPT-5.5's and Claude Opus 4.8's output tiers. The key is matching model capability to task complexity.

  • Simple classification, routing, and extraction — Small, fast models (GPT-5.4 mini, Gemini Flash-Lite, Claude Haiku 4.5) are sufficient and dramatically cheaper. Running a flagship model on tasks that don't require it wastes 10–50× on costs.
  • Complex reasoning, coding, and analysis — This is where flagship models earn their premium. GPT-5.5, Claude Opus 4.8, and Gemini 3.1 Pro significantly outperform cheaper tiers on tasks requiring multi-step reasoning, code generation, and nuanced analysis.
  • High-volume production workloads — The cheapest model that reliably achieves your quality bar is the correct choice. At scale, the cost difference between models can mean thousands of dollars per month. Always benchmark quality vs. cost before committing to a model for production.

Practical Tips for Reducing AI API Costs

Once you understand how pricing works, several strategies can meaningfully reduce your bill without sacrificing quality:

  • Prompt caching — Both Anthropic and OpenAI offer prompt caching, where repeated system prompts and context are cached and charged at a significantly reduced rate (typically 50–90% discount on cached tokens). If your system prompt is long and reused across many requests, caching can dramatically cut costs.
  • Batch processing — OpenAI's Batch API and Anthropic's Message Batches API offer 50% discounts on requests that don't need real-time responses. If you're running evaluations, offline analysis, or bulk content generation, batch mode cuts your bill in half.
  • Trim your context window — Ruthlessly remove unnecessary context from your prompts. Old conversation turns, repeated instructions, and verbose system prompts all add tokens that cost money on every request.
  • Use structured outputs — Asking models to respond in JSON with a defined schema reduces output verbosity, which reduces output tokens. A structured response is typically shorter than the equivalent prose response.

AI Model Pricing in 2026: What's Changed

AI API pricing has dropped dramatically over the past two years. GPT-4-class capability that cost $30–60 per million tokens in 2023 is now available for $2.50–10 per million tokens in 2026, with smaller distilled models delivering 80–90% of that quality for a fraction of the price. Competition between OpenAI, Anthropic, Google, and Meta has accelerated this deflation — and it shows no signs of stopping.

The emergence of reasoning models (OpenAI o-series, Claude's extended thinking mode) has added a new pricing dimension: "thinking tokens" used during the model's internal reasoning process before it produces a response. These models are significantly more expensive per request than standard models, but can be dramatically more accurate on complex reasoning tasks — making cost-per-correct-answer the relevant metric rather than cost-per-token.
